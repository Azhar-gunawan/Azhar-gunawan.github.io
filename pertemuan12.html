    Part 1: Packet Tracer - Navigating the IOS
    Part 1. Establish Basic Connections, Access the CLI, and Explore Help

    Langkah 1 : Menghubungkan PC 1 ke S1(Switch 1) menggunakan kabel konsol.
    a. Klik ikon Connections (ikon seperti petir) di pojok kiri bawah jendela packet tracer.

    b. Pilih kabel konsol berwarna biru muda dengan mengklik, pointer mouse akan menjadi apa yang dipilih.

    c. Klik PC1. Tampilan windows akan muncul pilihan untuk koneksi RS-232.

    d. Tarik dari ujung PC1 untuk mengkoneksikan kabel konsol dengan S1.

    e. Pilih Port Konsol untuk menyelesaikan koneksi.



    Langkah 2 : Buat sesi terminal dengan S1
    a. Klik PC 1 dan kemudian Pilih Tab Desktop


    b. Klik ikon aplikasi Terminal. Periksa bahwa pengaturan pada Konfigurasi Port sudah sesuai.

    c. Klik Oke

    Layar yang ditampilkan akan menampilkan beberapa pesan. Seperti pesan harus menekan tombol RETURN to get started. Tekan ENTER.

    Langkah 3 : Telusur Bantuan IOS
    a. IOS dapat memberikan bantuan seperti perintah tergantung pada level yang sedang diakses. Prompt yang sedang tampil saat ini adalaj User EXEC, dan sedang menunggu perintah untuk dimasukkan. Pada prompt ketikan symbol tanda tanya (?) untuk menampilkan daftar perintah.


    b. Pada prompt, ketik “t” lalu tanda tanya (?)

    c. Pada prompt, ketik “te” dan kemudian tanda tanya (?)

    Part 2. Explorer EXEC Mode

    Dalam bagian kedua ini, kita akan beralih ke mode hak khusus EXEC dan perintah tambahan.
    Langkah 1 : Masuk ke mode khusus
    a. Pada prompt , ketikan tanda tanya (?)

    b. Ketik “en” dan tekan tombol Tab

    Ini disebut juga dengan penyempurnaan perintah (penyempurnaan tab). Ketika bagian perintah diketik, Tab dapat digunakan untuk melengkapi Sebagian perintah. c. Masukkan perintah “enable” dan tekan ENTER.

    d. Pada prompt, ketikan tanda tanya Kembali (?)

    Langkah 2 : Masuk ke mode konfigurasi Global.
    a. Saat berada dalam mode EXEC yang khusus, perintah yang dimulai dengan huruf ‘C’ adalah configure. Ketikan perintah lengkap atau perintah Sebagian untuk membuatnya unik. Tekan tombol Tab untuk mengeluarkan perintah dan tekan ENTER.

    b. Tekan Enter untuk menerima parameter default diantara tanda kurung [terminal]

    c. Mode Konfigurasi global. Mode ini akan ditelusuri lebih lanjut dalam aktivitas kedepanya, untuk saat ini Kembali ke mode EXEC dengan mengetikan end, exit, atau CTRL+Z.

    Part 3. Set the Clock

    Langkah 1 : Menggunakan perintah “clock”.
    a. Gunakan perintah clock untuk melihat lebih lanjut bantuan dan sintaks dari perintah. Ketikan show clock pada prompt EXEC khusus.


    b. Gunakan perintah sensitive context dan perintah clock untuk mengatur waktu pada switch ke waktu saat ini. Masukkan perintah clock dan tekan ENTER.

    c. Muncul “%inclompete command”. Artinya kita perlu perintah parameter untuk perintah jam tersebut. Ketikan perintah clock dan tambahan tanda tanya (?)

    d. Atur jam menggunakan perintah clock set. Lanjutkan melalui perintah tersebut pada perintah waktu.

    e. Berdasarkan informasi yang diminta pada perintah clock set? Masukkan perintah waktu 19:00 P.M dengan menggunakan satuan format 24 Jam
    f. Ketikan clock set 19:00:00 31 Jan 2035. Lalu lihat jam yang sudah disetting dengan show clock.

    a. Perintah yang menampilkan statistic untuk semua antarmuka konfigurasi pada router


    b. Perintah yang menampilkan informasi tentang serial antarmuka 0/0/0 c. Masukkan perintah untuk menampilkan statistic antarmuka serial 0/0/0 pada R1
    1. Alamat Ip yang dikonfigurasi pada R1 -> 209.165.200.225/30
    2. Bandwith pada antarmuka serial 0/0/0 -> 1544 Kbits
    d. Masukkan perintah untuk menampilkan statistic antarmuka GigabitEthernet 0/0
    1. IP pada R1 untuk GigabitEthernet tidak ada konfigurasinya
    2. MAC address dari antarmuka GigabitEthernet 0/0 -> 1000000 Kbit
    Langkah 2 : Tampilkan daftar ringkasan antarmuka pada R1
    a. Perintah yang menampilkan ringkasan singkat dari interfaces saat ini, status, dan alamat ip


    b. Masukkan perintah pada setiap router
    1. Berapa banyak antarmuka serial yang ada di R1 dan R2 -> Setiap router memiliki 2 antarmuka serial
    2. Berapa banyak antarmuka Ethernet yang ada di R1 dan R2 -> R1 memiliki 6 antarmuka Ethernet sedangkan R2 hanya 2 antarmuka Ethernet
    3. Apakah semua antarmuka Ethernet pada R1 sama? -> Tidak. Ada 2 antarmuka Gigabit Ethernet dan 4 antarmuka Fast Ethernet. Antarmuka Gigabit Ethernet mendukung kecepatan hingga 1.000.000.000 bit dan antarmuka Fast Ethernet mendukung kecepatan hingga 1.000.000 bit.
    Langkah 3 : Tampilkan tabel routing R1
    a. Perintah apa yang menampilkan isi tabel routing

    b. Masukkan perintah pada R1
    1. Berapa banyak rute yang terhubung (menggunakan kode C) -> 1
    2. Rute yang terlist -> 209.165.200.224/30 dan 209.165.200.225/30
    3. Bagaimana cara router menangani paket yang ditujukan untuk jaringan yang tidak tercantum dalam tabel rute -> Router hanya akan mengirim paket ke jaringan yang terdaftar di tabel routing. Jika jaringan tidak terdaftar, paket akan keluar.

    Langkah 1 : Gunakan perintah verifikasi untuk memeriksa konfigurasi antarmuka
    a. Gunakan perintah show ip interface brief pada R1 dan R2
    - Berapa banyak antarmuka pada R1 dan R2? -> 3 pada setiap Router

    - Jika router mengetahui semua rute dalam jaringan, maka jumlah rute yang terhubung dan rute yang dipelajari secara dinamis (EIGRP) harus sama dengan jumlah total LAN dan WAN. Berapa banyak LAN dan WAN dalam topologi? 5
